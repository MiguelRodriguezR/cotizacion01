1. Resumen Ejecutivo

Se propone el desarrollo de una aplicación para la inspección, registro y análisis de fallas en pavimentos, orientada a empresas de ingeniería civil, contratistas de obra pública y organismos de control vial. La solución permitirá optimizar los procesos de inspección en campo, centralizar la información recopilada y generar reportes analíticos que faciliten la toma de decisiones para el mantenimiento y rehabilitación de infraestructura vial.

2. Objetivos del Proyecto

  • Digitalizar y optimizar el proceso de inspección de pavimentos en campo
  • Centralizar la información de múltiples proyectos y tramos viales
  • Facilitar el registro fotográfico y georreferenciado de fallas
  • Automatizar la generación de reportes y análisis estadísticos
  • Proporcionar una herramienta móvil-first accesible desde cualquier dispositivo
  • Mejorar la trazabilidad y documentación de inspecciones

3. Descripción Detallada de la Aplicación

Módulo 1: Dashboard Principal

Panel de control simple que presenta estadísticas consolidadas de las inspecciones registradas.

Funcionalidades:

  • Estadísticas generales: Número de fallas registradas, total del proyecto, kilómetros inspeccionados
  • Accesos directos: Botones para registrar falla, ver mapa y generar reporte
  • Lista de fallas recientes: Visualización de últimas fallas con nivel de severidad

Módulo 2: Gestión del Proyecto

Sistema básico para administrar el proyecto de inspección y sus tramos.

Funcionalidades:

  • Datos del proyecto: Nombre, ubicación, longitud total y abscisado
  • Gestión de tramos: División en tramos específicos (Ej: Km 0+000 a Km 5+000)
  • Progreso visual: Indicador simple del porcentaje inspeccionado

Módulo 3: Registro de Fallas (Core del Sistema)

Módulo principal para registrar fallas en pavimento desde dispositivos móviles Android.

3.1 Selección de Tramo

  • Selector del tramo donde se detectó la falla

3.2 Captura Fotográfica

  • Cámara del dispositivo: Acceso a la cámara del teléfono Android
  • Vista previa: Visualización de la foto capturada
  • Opción de recaptura: Posibilidad de tomar nuevamente la foto
  • Almacenamiento local: Guardado de imágenes en el dispositivo

3.3 Clasificación de la Falla

  • Tipo de falla: Catálogo según normativa técnica
    • Piel de Cocodrilo (Fatiga)
    • Fisura Longitudinal
    • Fisura Transversal
    • Fisura de Borde
    • Bache (Hueco)
    • Desintegración
    • Ahuellamiento
    • Corrugación
    • Pérdida de Agregados
    • Grieta de Reflexión
    • Otros (especificar)
  • Nivel de severidad:
    • Bajo
    • Medio
    • Alto
  • Ubicación específica: Carril Derecho, Carril Izquierdo, Carril Central, Berma, Separador

3.4 Geolocalización GPS

  • Coordenadas automáticas: Captura de latitud y longitud
  • Campo de abscisado: Ingreso de la abscisa (Ej: Km 12+450)
  • Funcionamiento offline: Registro sin conexión a internet

3.5 Mediciones de la Falla

  • Ingreso de longitud (metros)
  • Ingreso de ancho (metros)
  • Cálculo automático de área (m²)

3.6 Observaciones

  • Campo de texto para notas adicionales
  • Condiciones climáticas (opcional)

3.7 Guardado

  • Almacenamiento local en el dispositivo
  • Confirmación del registro exitoso

Módulo 4: Mapa de Fallas

Visualización de fallas registradas en un mapa interactivo.

Funcionalidades:

  • Mapa básico: Visualización con OpenStreetMap (sin costos)
  • Marcadores por severidad:
    • 🔴 Rojo: Severidad Alta
    • 🟠 Naranja: Severidad Media
    • 🟢 Verde: Severidad Baja
  • Ventana de información: Al tocar un marcador muestra tipo, severidad y foto
  • Controles básicos: Zoom y centrado en ubicación actual

Módulo 5: Dashboard de Análisis

Visualización consolidada de todas las fallas con estadísticas y gráficos para presentación del proyecto universitario.

5.1 Estadísticas Generales

  • Resumen numérico:
    • Total de fallas registradas
    • Distribución por severidad (cantidad y porcentaje)
    • Área total afectada (m²)
    • Kilómetros inspeccionados

5.2 Visualizaciones Gráficas

  • Gráfico de barras: Distribución de fallas por severidad
  • Gráfico de pastel: Distribución por tipo de falla
  • Tabla de datos: Listado de tipos de falla ordenados por frecuencia
  • Análisis por carril: Comparativa de afectación entre carriles

5.3 Exportación Básica

  • Captura de pantalla del dashboard: Para incluir en presentación
  • Exportación de datos CSV: Tabla con todas las fallas para procesamiento en Excel si se requiere

4. Características Técnicas

4.1 Aplicación Android Offline

  • App instalable en dispositivos Android (APK)
  • Funcionamiento 100% offline (sin necesidad de internet en campo)
  • Almacenamiento local de datos y fotografías
  • Base de datos local SQLite
  • Exportación de datos para consolidación posterior

4.2 Interfaz Optimizada para Campo

  • Diseño simple y directo para uso en inspección
  • Botones grandes para facilitar uso con guantes
  • Validación de campos esenciales
  • Formulario en una sola pantalla con scroll

5. Stack Tecnológico Propuesto

App Móvil React Native o Flutter para desarrollo Android
Base de Datos Local SQLite para almacenamiento offline en el dispositivo
Dashboard librería de gráficos (Chart.js)
Mapas OpenStreetMap / Leaflet (gratuito, sin límites de uso)
Almacenamiento Local en dispositivo (no requiere servidor)
Exportación de datos CSV para compartir datos entre usuarios
Nota sobre arquitectura: Al ser un proyecto universitario de corta duración, la aplicación no requiere servidor ni base de datos centralizada. Cada usuario registra datos en su dispositivo y estos se pueden consolidar posteriormente para el dashboard de presentación.
Nota: El stack tecnológico final será definido en conjunto con el cliente según sus preferencias, infraestructura existente y requisitos específicos de integración.
